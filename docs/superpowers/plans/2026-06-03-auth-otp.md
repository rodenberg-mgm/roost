# Auth OTP Code Entry — Implementation Plan

> Execute with superpowers:subagent-driven-development, task-by-task, on a branch off `main` (e.g. `feat/auth-otp`). Steps use `- [ ]` checkboxes.

**Goal:** Add a 6-digit code path to the login and join flows (typed code via `verifyOtp`), keeping the magic link as a fallback.

**Spec:** `docs/superpowers/specs/2026-06-03-auth-otp-design.md`

**Prereq (Matt, dashboard, before e2e):** add `{{ .Token }}` to the Supabase **Magic Link** email template so the email shows the 6-digit code (see spec).

**Tech:** Next.js 16 App Router, `@supabase/ssr` browser client (`lib/supabase/client.ts` → `createClient()`), TypeScript. No new deps.

---

## Key facts (from the codebase)

- Browser client: `import { createClient } from "@/lib/supabase/client";` → `createClient().auth.signInWithOtp(...)` / `.verifyOtp(...)`. `@supabase/ssr`'s browser client persists the session to cookies, so after a client-side `verifyOtp` the server (middleware, server components) sees the session.
- Login form: `app/(auth)/login/login-form.tsx` — calls `signInWithOtp({ email, options: { emailRedirectTo: `${origin}/auth/callback` } })`, then shows a "check your email" screen. Success destination: `/dashboard`.
- Join form: `app/trip/[token]/join/join-form.tsx` — calls `signInWithOtp({ email, options: { emailRedirectTo: `${origin}/auth/callback?next=/trip/${token}/join/complete`, data: { display_name: name } } })`. Success destination: `/trip/${token}/join/complete` (runs `consumeInviteAndJoin`).
- Magic-link callback `app/(auth)/auth/callback/route.ts` (`exchangeCodeForSession`) stays unchanged — the link remains a fallback.
- `verifyOtp` for an email code: `supabase.auth.verifyOtp({ email, token, type: "email" })`.

## Task 1: Shared OTP code-entry component

**Create:** `components/auth/otp-code-step.tsx`

A presentational + verify step shared by both forms. It does NOT send the email (the parent already called `signInWithOtp`); it only verifies a typed code and reports resend/back.

- [ ] **Step 1: Write the component**

```tsx
"use client";

import { Button } from "@/components/ui/button";
import { Input } from "@/components/ui/input";
import { Label } from "@/components/ui/label";
import { createClient } from "@/lib/supabase/client";
import { Loader2, Mail } from "lucide-react";
import { useState } from "react";

interface OtpCodeStepProps {
  email: string;
  /** Called after a successful verifyOtp (session is now set). */
  onVerified: () => void;
  /** Re-send the code (parent re-calls signInWithOtp with the same options). */
  onResend: () => Promise<void>;
  /** Go back to the email/name step. */
  onChangeEmail: () => void;
}

export function OtpCodeStep({ email, onVerified, onResend, onChangeEmail }: OtpCodeStepProps) {
  const [code, setCode] = useState("");
  const [verifying, setVerifying] = useState(false);
  const [resending, setResending] = useState(false);
  const [error, setError] = useState<string | null>(null);

  async function handleVerify(e: React.FormEvent) {
    e.preventDefault();
    setVerifying(true);
    setError(null);

    const supabase = createClient();
    const { error: vErr } = await supabase.auth.verifyOtp({
      email,
      token: code.trim(),
      type: "email",
    });

    if (vErr) {
      setVerifying(false);
      setError("That code didn't match. Check the email or resend a new one.");
      return;
    }
    // Session cookie is now set by the ssr browser client; let the parent navigate.
    onVerified();
  }

  async function handleResend() {
    setResending(true);
    setError(null);
    try {
      await onResend();
    } finally {
      setResending(false);
    }
  }

  return (
    <form onSubmit={handleVerify} className="rounded-card border bg-card p-6 text-center shadow-card">
      <div className="mx-auto mb-4 flex h-12 w-12 items-center justify-center rounded-full bg-forest/10">
        <Mail className="h-6 w-6 text-forest" />
      </div>
      <h2 className="text-lg font-semibold text-ink">Check your email</h2>
      <p className="mt-2 text-sm text-ink-light">
        Enter the 6-digit code we sent to <strong>{email}</strong>, or just click the link in the email.
      </p>

      <div className="mt-5 space-y-2 text-left">
        <Label htmlFor="otp-code">6-digit code</Label>
        <Input
          id="otp-code"
          inputMode="numeric"
          autoComplete="one-time-code"
          maxLength={6}
          value={code}
          onChange={(e) => setCode(e.target.value.replace(/\D/g, ""))}
          placeholder="123456"
          autoFocus
          className="text-center text-lg tracking-[0.4em]"
        />
      </div>

      {error && <p className="mt-3 text-sm text-red-600">{error}</p>}

      <Button
        type="submit"
        className="mt-4 w-full bg-forest text-white hover:bg-forest-dark"
        disabled={verifying || code.length < 6}
      >
        {verifying ? (
          <>
            <Loader2 className="mr-2 h-4 w-4 animate-spin" />
            Verifying…
          </>
        ) : (
          "Verify & continue"
        )}
      </Button>

      <div className="mt-4 flex items-center justify-between text-xs text-ink-light">
        <button type="button" onClick={onChangeEmail} className="transition-colors hover:text-forest">
          ← Change email
        </button>
        <button
          type="button"
          onClick={handleResend}
          disabled={resending}
          className="transition-colors hover:text-forest disabled:opacity-50"
        >
          {resending ? "Resending…" : "Resend code"}
        </button>
      </div>
    </form>
  );
}
```

- [ ] **Step 2: Typecheck** — `npx tsc --noEmit`. Confirm `<Input>` accepts `inputMode`/`autoComplete`/`maxLength` (it spreads props). Confirm `verifyOtp` signature compiles.
- [ ] **Step 3: Commit** — `feat(auth): shared OTP code-entry step component`

## Notes for the implementer

- The session is set **client-side** by `verifyOtp` (the ssr browser client writes cookies); navigating to a server route afterward is what makes the server see it. Use `router.push` (+ `router.refresh()` on login) — don't call any server `exchangeCodeForSession` for the code path.
- Keep `shouldCreateUser` default (true) so first-time guests are created; the `data.display_name` must remain on the join `signInWithOtp` call.
- Don't touch `app/(auth)/auth/callback/route.ts` — the magic link stays as the fallback.
- If `verifyOtp` rejects with "Token has expired or is invalid," that's the wrong/expired code path — the generic error copy covers it.
