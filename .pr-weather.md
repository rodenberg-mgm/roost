## What

A **Weather** card on the trip guide showing the forecast for the trip's dates + public location, via the keyless [Open-Meteo](https://open-meteo.com) API. Sits in the info stack right after Location.

Horizontal-scroll day row: weekday, condition icon (Lucide, WMO-code mapped), hi/lo (°F), and precip % when ≥30.

## Graceful states (the card never breaks the page)

- **TBD dates** → host-only "Set trip dates to see the forecast" hint; nothing for guests
- **>2 weeks out** → "The forecast opens about two weeks before your trip"
- **Past trip / no city / any API failure** → renders nothing
- Forecast window clamped to `[today, today+15]` (Open-Meteo serves ~16 days)

## Implementation

- `lib/weather/open-meteo.ts` — server-only geocode + forecast orchestrator. Never throws; returns a discriminated `TripWeather` status. `fetch` cached via `next.revalidate` (geocode 24h, forecast 1h). No API key, no new deps.
- `lib/weather/weather-code.ts` — WMO code → label + Lucide icon.
- `components/trip-weather.tsx` — presentational client card.
- `app/(app)/trips/[id]/page.tsx` — resolves weather from the trip's existing `city`/`region`/`starts_on`/`ends_on` and renders the section conditionally.

Geocoding uses only the **public** city/region — never the sensitive address.
